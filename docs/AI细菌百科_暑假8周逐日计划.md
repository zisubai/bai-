# AI 细菌百科 · 暑假 8 周逐日计划

> 目标:用两个月,从零基础到做出一个**能在浏览器里搜索细菌、显示菌种卡片、用 AI 生成摘要**的真实小产品,并完成一次 10 分钟路演。
> 工具:学习用 **Codex**(OpenAI 智能体编程工具);项目里的 AI 功能调用 **DeepSeek API**。
> 节奏:周一到周五学习(每天约 2–3 小时),周六演示+写 changelog,周日休息。

---

## 〇、开工前必读:把 Codex 当家教,不是当替身

这是整份计划里**最重要的一页**。Codex 能自动写代码,如果放任它,孩子会变成"只会按回车接受 diff、却什么都不懂"的人——那这个暑假就白费了。所以立六条铁律:

1. **默认用 suggest(建议)模式**。Codex 每改一个文件、每跑一条命令都要你确认。绝不开 full-auto。看不懂的 diff,不准点同意。
2. **前 3 周核心逻辑自己写,卡住再问**。Codex 只用来搭脚手架(建项目、装依赖、生成模板)和**讲解**。真正的循环、判断、数据结构,自己敲。
3. **每段 Codex 写的代码,都要能讲出来**。养成习惯:让它写完后,问一句「逐行解释这段代码,我是初学者」。讲不出来的代码,等于没学会。
4. **把 Codex 当老师问思路,而不是直接要答案**。好提示词是:「我想实现 X,先别写代码,给我思路和步骤,我自己试」。
5. **每天写一行 changelog**:"今天能跑了、昨天还不行的是什么"。一句话,但能攒出"我在进步"的实感,也是给爸爸的进度凭证。
6. **周六演示,加一条**:除了"只看运行结果",每两周让他**不看笔记、用大白话给你讲一遍这东西怎么跑起来的**。这是检验 Codex 有没有替他思考的唯一办法。

## 阶段一(第 1–2 周):打地基 + 第一个能跑的命令行版

**本阶段目标**:学会终端、Git/GitHub、Python 基础;把项目仓库建起来;做出**命令行版细菌百科**(能录入、查询 3–5 种真实细菌)。

### 第 1 周:工具 + 项目骨架

- **周一｜环境与终端**
  - 学:命令行基础(`cd` / `ls` / `mkdir` / 文件路径概念)
  - 练:配置环境(见上"一次性配置");在 Codex 里问「带我用命令行新建一个叫 bacteria-wiki 的文件夹并进去」,**看着它做,自己复述每条命令什么意思**
  - 产出:环境全部装好,能进入项目文件夹

- **周二｜Git 与 GitHub**
  - 学:Git 是什么、commit / push / 仓库的概念(廖雪峰 Git 教程前 1/3)
  - 练:在 Codex 里把 bacteria-wiki 初始化成 git 仓库,在 GitHub 建一个空仓库,关联并 push 一个空 README。提示词:「带我把这个文件夹变成 git 仓库,并推送到我的 GitHub,每一步解释」
  - 产出:GitHub 上能看到自己的仓库

- **周三｜Python 语法 ①**
  - 学:变量、字符串、列表、字典、`print`、`if`
  - 练:**自己手写**一个 `hello.py`,打印 3 种细菌的名字。卡住才问 Codex,但**不准让它整段代写**
  - 产出:能跑出三行细菌名

- **周四｜Python 语法 ②**
  - 学:`for` 循环、函数 `def`、从命令行 `input()` 读输入
  - 练:自己写一个小程序:用户输入一个细菌名,程序回一句固定介绍。Codex 仅在卡住时讲思路
  - 产出:一个能"问答"的小脚本

- **周五｜数据结构:把细菌建模**
  - 学:用"字典"表示一条细菌(名字、革兰氏染色、形状、栖息地、是否致病)
  - 练:和 Codex 讨论:「细菌百科里一条细菌应该有哪些字段?帮我设计一个字典结构,先别写完整程序」。然后自己把 **3 种真实细菌**(如大肠杆菌 E. coli、枯草芽孢杆菌 B. subtilis、金黄色葡萄球菌 S. aureus)的真实数据填进去(数据查 NCBI / 维基)
  - 产出:一个含 3 条真实细菌的数据文件

- **周六｜演示 + 收尾**
  - 给爸爸演示:命令行输入细菌名 → 输出它的信息。讲不讲概念无所谓,**要能跑**
  - 写本周 changelog;把代码 push 到 GitHub

### 第 2 周:命令行版做完整 + 项目管理上手

- **周一｜项目结构与模块**
  - 学:为什么要把代码拆成多个文件(`main.py` / `data.py`);`import` 是什么
  - 练:让 Codex 帮你把上周的代码拆成 `data.py`(存细菌数据)和 `main.py`(查询逻辑),**它每拆一步你要看懂**
  - 产出:结构清晰的两文件项目

- **周二｜命令行交互菜单**
  - 学:`while` 循环做菜单(1.查询 2.列出全部 3.退出)
  - 练:自己写主循环框架,菜单分支让 Codex 补;反复测试边界(输入不存在的细菌怎么办?)
  - 产出:一个有菜单、能反复操作的命令行程序

- **周三｜读写文件:JSON**
  - 学:什么是 JSON;Python 怎么读写 JSON 文件
  - 练:把细菌数据从代码里搬到 `bacteria.json`,程序启动时读取。提示词:「带我把硬编码的数据改成从 json 文件读取,解释 json 的好处」
  - 产出:数据和代码分离,新增细菌只改 json

- **周四｜GitHub 项目管理**
  - 学:GitHub 的 **Issues**(任务清单)和 **Projects**(看板)怎么用;什么是好的 commit message
  - 练:在 GitHub 上给项目建一个看板,把"接下来想做的功能"拆成几条 Issue(如:加搜索、加更多细菌、做网页版)。这就是**软件工程的需求管理**
  - 产出:一个有待办看板的项目

- **周五｜加功能:模糊搜索 + 扩充数据**
  - 学:字符串包含判断(`in`)、列表筛选
  - 练:自己实现"输入一部分名字也能搜到";把细菌扩到 **5–8 种**(真实数据)
  - 产出:命令行版基本完整

- **周六｜阶段演示 + 复盘**
  - 给爸爸完整演示命令行版;**这次让他用大白话讲一遍程序是怎么读 json、怎么搜的**(检验学习真实性)
  - changelog;push;在看板上把已完成的 Issue 关掉

  - 产出:像个产品的界面

  - 练:完善 README(截图、功能列表、技术栈、怎么运行);这就是将来给大学/实验室看的作品集
  - 产出:专业的项目主页

- **周四｜准备 10 分钟路演**
  - 内容必须包含:**①用户是谁(给谁用的)②解决什么问题 ③现场 live demo ④技术栈一句话 ⑤这两个月我学到了什么**
  - 练:写讲稿、掐时间、自己演练 2 遍
  - 产出:一份能讲的路演

- **周五｜彩排**
  - 对着爸爸/家人完整跑一遍 10 分钟路演,收集反馈、最后微调
  - 产出:路演定稿

- **周六｜正式路演 🎉**
  - 完整演示 + 讲解;录屏存档;写最后一篇 changelog:"两个月前我连命令行都不会"
  - **庆祝。这是一个真实交付的项目。**

---

## 四、如果超前 / 落后了,怎么调

- **落后了**(很正常):**砍 RAG**(第7周冲刺部分)是第一刀;再不行,前端简化成"够演示就行";**绝不砍掉"每周能跑+演示"这条主线**。能跑的小东西 > 做一半的大东西。
- **超前了**(说明是块料):优先加**更多真实细菌数据**和**数据可视化**(用 pandas 画致病菌分布图);再有余力才碰 RAG 或把项目**部署上线**(让别人用网址也能访问)。
- **某天卡死超过 1 小时**:停下来,让 Codex「用初学者能懂的方式,把这个错误拆成几步带我排查」。排查能力比写代码更重要。
